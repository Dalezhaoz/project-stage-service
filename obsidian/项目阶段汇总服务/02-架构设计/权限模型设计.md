# 权限模型设计

## 角色

### admin

权限最高，负责：

- 用户管理
- 服务器配置
- 中心库配置
- 调度配置
- 主钉钉配置
- 查看全部项目
- 修改全部项目元数据

### internal

内部业务或运维人员，负责：

- 查看全部项目
- 管理服务器配置
- 测试 Agent
- 修改负责人
- 修改应用服务器分配
- 管理应用服务器候选项

### external

外部协作人员，仅能：

- 查看自己负责的项目
- 修改自己项目的应用服务器分配
- 在管理员允许时手工刷新

## 当前认证方式

- Cookie Authentication

## 当前策略

- `AdminOnly`
- `InternalOrAbove`

## 当前数据级权限

external 用户访问查询时，不是前端裁剪，而是服务端通过 `project_metadata` 中的：

- `server_name`
- `exam_code`
- `maintainer`

组合过滤项目范围。
