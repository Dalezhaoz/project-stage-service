# 总体架构

## 组件划分

### 1. 前端看板

位置：

- `wwwroot/index.html`

职责：

- 登录与初始化账号
- 查询筛选
- 看板展示
- 项目负责人维护
- 应用服务器分配
- 顶部统计摘要

### 2. 主服务

位置：

- `Program.cs`
- `Services/`

职责：

- 用户认证与权限
- 服务器配置管理
- 中心库配置管理
- 调度管理
- 调 Agent 测试与刷新
- 统一写中心表
- 中心表查询
- Excel 导出
- 钉钉通知

### 3. Agent

位置：

- Debian：`tools/stage_agent/stage_agent.py`
- Windows：`tools/stage_agent_win/`

职责：

- 接收主服务加密任务
- 使用本机网络访问源库
- 执行主服务下发的查询定义
- 返回标准化结果

### 4. 中心库

类型：

- SQL Server

职责：

- 保存标准化阶段记录
- 保存项目负责人与应用服务器分配
- 保存应用服务器候选项
- 作为页面唯一查询来源

### 5. 本地缓存

类型：

- SQLite

职责：

- 保存主服务最近一次快照
- 支持缓存状态查看
- 便于故障排查

## 真实数据流

```text
前端 -> 主服务 -> Agent -> 源数据库
前端 <- 主服务 <- 中心表 <- 主服务
```

展开后是：

1. 用户在页面点击“更新数据”
2. 主服务读取启用的服务器配置
3. 主服务按服务器调用对应 Agent
4. Agent 本地执行查询并返回标准化结果
5. 主服务聚合所有 Agent 返回结果
6. 主服务写入中心表
7. 主服务将快照保存到本地 SQLite
8. 页面再从中心表查询展示

## 当前设计原则

- 主服务负责业务规则
- Agent 负责本地执行
- 中心表负责统一查询
- 权限过滤在服务端完成
- 源库账号密码不以明文通过 HTTP 传输
