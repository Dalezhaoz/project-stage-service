# 中心表设计

## 中心库类型

- SQL Server

## 主表列表

- `dbo.project_stage_summary`
- `dbo.project_metadata`
- `dbo.app_server_options`

## 设计目标

- 将多源 SQL Server / MySQL 结果统一标准化
- 作为前端页面唯一查询来源
- 避免页面实时打源库
- 承接项目负责人和应用服务器分配

## 当前写入方式

1. 主服务从 Agent 收到标准化结果
2. 主服务构建 `ProjectStageSummary`
3. 主服务调用 `SummaryStoreService.SyncSnapshotAsync`
4. 按 `source_server_name + source_database_name` 分组清理旧数据
5. 去重后写入新数据

## 当前读取方式

页面相关接口统一从中心表读取：

- `/api/query`
- `/api/stages`
- `/api/export`
- `/api/board-counts`

## 唯一键

当前逻辑唯一性依据：

- `source_server_name`
- `source_database_name`
- `exam_code`
- `stage_name`
- `stage_start_time`
- `stage_end_time`
