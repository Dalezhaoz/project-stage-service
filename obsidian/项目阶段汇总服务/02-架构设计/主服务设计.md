# 主服务设计

## 技术栈

- ASP.NET Core Minimal API
- Cookie Authentication
- SQL Server 中心库
- SQLite 本地缓存
- ClosedXML 导出

## 主服务核心职责

### 1. 认证与权限

主要服务：

- `LocalAuthService`

负责：

- 初始化管理员账号
- 登录校验
- 首次登录强制改密码
- 用户角色管理
- 用户级钉钉配置
- 是否允许普通用户手工刷新

### 2. 配置管理

主要服务：

- `ServerConfigStore`
- `SummaryStoreConfigStore`
- `ScheduleConfigStore`

负责：

- 源服务器配置
- 中心库配置
- 定时任务配置

### 3. Agent 调用

主要服务：

- `AgentClientService`
- `AgentPayloadProtector`

负责：

- `/test` 调用 agent 健康与数据库可达性测试
- `/query` 调用 agent 返回标准化结果
- 使用 AES-GCM 加密传输源库和中心库凭据

### 4. 刷新与中心表写入

主要服务：

- `ProjectStageRefreshService`
- `ProjectStageCountRefreshService`
- `SummaryStoreService`
- `ProjectStageSummaryBuilder`

负责：

- 调 Agent 获取阶段与人数
- 聚合为 `ProjectStageSummary`
- 回写 SQLite
- 写入中心表

### 5. 页面查询与导出

主要服务：

- `SummaryStoreService`
- `ProjectStageExportService`
- `ProjectMetadataService`

负责：

- 按筛选条件查询中心表
- 按项目负责人做服务端过滤
- 导出 Excel

## 当前已经取消的模式

- 主服务直连源数据库作为正式刷新模式

保留的类和逻辑可能仍有部分历史痕迹，但正式运行链路应以：

- Agent -> 主服务 -> 中心表

为准。
