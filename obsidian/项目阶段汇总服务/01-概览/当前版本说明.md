# 当前版本说明

## 当前真实架构

当前系统已经不是“主服务直连所有源库”的模式，而是：

1. 主服务保存服务器配置、中心库配置、调度配置和用户权限
2. 主服务向各业务服务器上的 Agent 发起加密请求
3. Agent 在本机访问 SQL Server 或 MySQL 源库
4. Agent 返回标准化结果给主服务
5. 主服务统一写入中心 SQL Server 表 `project_stage_summary`
6. 页面查询、导出、阶段筛选、看板统计全部读取中心表

## 当前关键特征

- 查询来源：中心表
- 刷新来源：Agent
- 本地缓存：SQLite 快照
- 权限模型：`admin / internal / external`
- 传输安全：主服务与 Agent 之间使用 AES-GCM 加密
- 项目维度权限：external 用户只看自己负责的项目

## 当前默认调度

- 阶段刷新：04:00
- 人数刷新：06:00
- 钉钉通知：08:00

## 当前重要约束

- 每台启用的业务服务器必须配置：
  - Agent 端口
  - Agent 密钥
- 中心库必须启用，否则刷新不可执行
- Agent 机器必须本地能访问自己的源数据库
